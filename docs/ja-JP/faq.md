## 接続に関する問題

### Q: 管理画面にアクセスできない

**A:** 以下を確認してください：

1. **URL が正しいか** — `https://your-domain.example.com` のように正しいアドレスを使用
2. **サーバーが起動しているか** — `npm start` が正常に動作しているか確認
3. **ポートがブロックされていないか** — ファイアウォール設定を確認
4. **IP制限が設定されていないか** — 管理者に確認

### Q: 「Offline」と表示される

**A:** WebSocket接続が切断されています：

1. ネットワーク接続を確認
2. ブラウザをリロード（F5）
3. サーバーのログを確認

### Q: 接続が頻繁に切断される

**A:**

- プロキシ/ロードバランサーのWebSocketタイムアウト設定を確認
- Tailscale使用時はserveモードの設定を確認
- ブラウザの拡張機能が干渉していないか確認

## チャネルに関する問題

### Q: チャネルが「未設定」のままになる

**A:**

1. 設定ファイル（`config.yaml`）のチャネル設定を確認
2. APIキー/トークンが正しいか確認
3. 「Probe」ボタンで接続テスト

### Q: WhatsApp の QR コードが表示されない

**A:**

1. WhatsAppチャネルが有効になっているか確認
2. サーバーがインターネットに接続できるか確認
3. 「QR表示」ボタンを再度クリック
4. ログに表示されるエラーメッセージを確認

### Q: WhatsApp の接続が切れた

**A:**

1. 「再連携」ボタンをクリックしてQRコードを再表示
2. スマートフォンでQRコードをスキャン
3. それでも解決しない場合は「ログアウト」→ 再連携を試行

### Q: Slack ボットが応答しない

**A:**

1. Slackアプリの設定でSocket Modeが有効か確認
2. `xapp-` で始まるApp-Level Tokenが正しいか確認
3. ボットに必要な権限（スコープ）が付与されているか確認
4. Channelsタブで状態を確認し「Probe」で接続テスト

### Q: Discord ボットが応答しない

**A:**

1. Botトークンが正しいか確認
2. Discordの開発者ポータルでIntentsが有効か確認（Message Content Intent等）
3. ボットがサーバーに招待されているか確認

### Q: Telegram ボットが応答しない

**A:**

1. BotFatherから取得したトークンが正しいか確認
2. Webhookモードの場合、公開URLが正しいか確認
3. Pollingモードの場合、他のプロセスでポーリングしていないか確認

## 認証に関する問題

### Q: パスワードが通らない

**A:**

1. パスワードの大文字・小文字を確認
2. 環境変数 `OPENCLAW_GATEWAY_PASSWORD` の値を確認
3. config.yaml の `gateway.auth.password` を確認
4. Rate Limitに引っかかっている場合は少し待ってから再試行

### Q: トークン認証でエラーになる

**A:**

1. Bearer トークンの形式が正しいか確認
2. 環境変数 `OPENCLAW_GATEWAY_TOKEN` の値を確認
3. トークンに余分な空白や改行がないか確認

### Q: SSO/SAMLでログインできない

**A:**

1. IdPの設定（エントリポイントURL、証明書）が正しいか確認
2. SP Entity IDがゲートウェイのURLと一致しているか確認
3. `allowedDomains` にメールドメインが含まれているか確認
4. IdP側のユーザー属性マッピングを確認

## チャットに関する問題

### Q: AIからの応答がない

**A:**

1. LLMプロバイダーのAPIキーが設定されているか確認
2. APIキーの残高/クォータを確認
3. エージェント設定でモデルが正しく選択されているか確認
4. ログでエラーメッセージを確認

### Q: 応答が途中で止まる

**A:**

- モデルのトークン制限に達している可能性があります
- ネットワークタイムアウトの可能性があります
- ログを確認してエラーの詳細を確認

### Q: 応答が遅い

**A:**

1. モデルの選択を確認（大きいモデルほど遅い）
2. ネットワーク遅延を確認
3. サーバーのリソース（CPU/メモリ）を確認

## 設定に関する問題

### Q: 設定の変更が反映されない

**A:**

1. 「保存」ボタンを確実にクリックしたか確認
2. 設定の適用（Apply）が必要な場合があります
3. 一部の設定はサーバーの再起動が必要です

### Q: 設定のバリデーションエラー

**A:**

- エラーメッセージの内容を確認
- YAML の構文が正しいか確認（インデント、コロン後のスペース等）
- 必須フィールドが入力されているか確認

### Q: 日本語が表示されない

**A:**

1. 画面右下の言語メニューから「日本語」を選択
2. ブラウザをリロード
3. ブラウザの言語設定で日本語が優先になっているか確認

## パフォーマンスに関する問題

### Q: 管理画面の表示が遅い

**A:**

1. ブラウザのキャッシュをクリア
2. ブラウザの開発者ツールでネットワークエラーを確認
3. サーバーのリソース使用率を確認

### Q: メモリ使用量が増え続ける

**A:**

- セッションの蓄積が原因の可能性があります
- 古いセッションを定期的に削除してください
- 監査ログの `maxEvents` 設定を調整してください

## その他

### Q: ライセンスについて

OpenClawはMITライセンスで提供されています。Fork元（OpenClaw）の著作権表示を保持する必要があります。詳細は `LICENSE` ファイルを参照してください。

### Q: アップデート方法

```bash
git pull origin main
npm install
npm run build
```

### Q: ログの出力先

- 管理画面の「Logs」タブでリアルタイム確認
- 監査ログが有効な場合、設定したファイルパスに出力
- 標準出力（stdout）にも出力されます

### Q: サポートへの問い合わせ

以下の情報を添えてお問い合わせください：

1. **OpenClawのバージョン** — 管理画面右上に表示
2. **ブラウザ情報** — ブラウザ名とバージョン
3. **エラーメッセージ** — 画面に表示されたエラーまたはログの内容
4. **再現手順** — 問題が発生する操作手順

**問い合わせ先:**

- GitHub Issues: https://github.com/preferred-inc/openclaw/issues
- メール: （管理者に確認してください）
